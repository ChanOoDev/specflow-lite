# GitHub Spec Kit Skill

## Purpose

Ensure specification-driven development.

## Workflow

Requirement
→ Clarification
→ Specification
→ Plan
→ Tasks
→ Implementation
→ Review
→ Test
→ Deploy

## Rules

### Never

- Code before specification
- Implement before planning
- Skip requirement discovery

### Always

- Clarify requirements
- Create specification
- Create plan
- Create tasks
- Review implementation

## GitHub Spec Kit Commands

### Specify

Generate requirements and specification.

### Plan

Generate implementation plan.

### Tasks

Generate actionable tasks.

### Implement

Execute tasks.
